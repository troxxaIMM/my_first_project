# Ответы на вопросы по Git

## 1. Какие три основные области работы с файлами в Git и чем они отличаются?

**Рабочая папка (Working Directory)** - это текущее состояние файлов в проекте, где вы непосредственно работаете с кодом.

**Индекс (Staging Area)** - промежуточная область, куда добавляются файлы перед коммитом. Здесь вы формируете "снимок" изменений для следующего коммита.

**Репозиторий (Repository)** - база данных Git, хранящая всю историю проекта, коммиты, ветки и метаданные.

## 2. Что происходит при выполнении команд:

**git add** - добавляет изменения из рабочей папки в индекс (staging area).

**git commit** - создает новый коммит из проиндексированных изменений и сохраняет его в локальный репозиторий.

**git push** - отправляет коммиты из локального репозитория в удаленный репозиторий (например, на GitHub).

## 3. Почему важно писать информативные сообщения к коммитам?

Информативные сообщения:
- Помогают понять историю изменений
- Облегчают поиск конкретных изменений
- Помогают новым разработчикам разобраться в проекте
- Упрощают отладку при появлении ошибок
- Позволяют автоматически генерировать changelog

## 4. Чем отличается `git add имя_файла` от `git add .`?

**`git add имя_файла`** - добавляет в индекс только указанный файл.

**`git add .`** - добавляет все измененные и новые файлы в текущей директории и поддиректориях.

**Когда использовать:**
- `git add имя_файла` - когда нужно добавить конкретные изменения
- `git add .` - когда все изменения связаны и должны быть в одном коммите

## 5. Что показывает команда `git status` и как меняется ее вывод?

`git status` показывает:
- Измененные, но не добавленные в индекс файлы
- Файлы в индексе, готовые к коммиту
- Неотслеживаемые файлы

**Изменения вывода:**
- После изменений: показывает modified files
- После `git add`: показывает changes to be committed
- После `git commit`: показывает "nothing to commit, working tree clean"

## 6. Для чего нужна команда `git remote add origin URL` и что означает "origin"?

Команда связывает локальный репозиторий с удаленным репозиторием.

**"origin"** - это стандартное имя для удаленного репозитория-источника. Это псевдоним для URL удаленного репозитория.

## 7. Зачем нужен флаг `-u` в команде `git push -u origin main`?

Флаг `-u` (или `--set-upstream`) устанавливает связь между локальной веткой и удаленной веткой. После этого можно использовать просто `git push` без указания репозитория и ветки.

## 8. Что такое ветка по умолчанию и какая у вас создалась?

**Ветка по умолчанию** - это основная ветка проекта, куда по умолчанию попадают изменения.

У меня создалась ветка **main**. Это важно потому, что:
- Это основная ветка разработки
- На нее обычно настроены процессы CI/CD
- Она считается стабильной версией проекта

## 9. Как просмотреть историю коммитов и какую информацию вы видите?

**Команда:** `git log`

**Информация:**
- Хеш коммита (уникальный идентификатор)
- Автор коммита
- Дата и время коммита
- Сообщение коммита

## 10. Что произойдет, если сделать изменения в файле, но не выполнить `git add` перед `git commit`?

Изменения не попадут в коммит. Git создаст коммит только из тех файлов, которые были добавлены в индекс с помощью `git add`. Непроиндексированные изменения останутся в рабочей директории.
