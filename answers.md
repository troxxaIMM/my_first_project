# Ответы на вопросы по Git

## 1. Какие три основные области работы с файлами в Git и чем они отличаются?

**Рабочая папка (Working Directory)** - это текущее состояние файлов в проекте, где вы непосредственно работаете с кодом.

**Индекс (Staging Area)** - промежуточная область, куда добавляются файлы перед коммитом. Здесь вы формируете "снимок" изменений для следующего коммита.

**Репозиторий (Repository)** - база данных Git, хранящая всю историю проекта, коммиты, ветки и метаданные.

## 2. Что происходит при выполнении команд:

**git add** - добавляет изменения из рабочей папки в индекс (staging area).

**git commit** - создает новый коммит из проиндексированных изменений и сохраняет его в локальный репозиторий.

**git push** - отправляет коммиты из локального репозитория в удаленный репозиторий (например, на GitHub).

## 3. Почему важно писать информативные сообщения к коммитам?

Информативные сообщения:
- Помогают понять историю изменений
- Облегчают поиск конкретных изменений
- Помогают новым разработчикам разобраться в проекте
- Упрощают отладку при появлении ошибок
- Позволяют автоматически генерировать changelog

## 4. Чем отличается `git add имя_файла` от `git add .`?

**`git add имя_файла`** - добавляет в индекс только указанный файл.

**`git add .`** - добавляет все измененные и новые файлы в текущей директории и поддиректориях.

**Когда использовать:**
- `git add имя_файла` - когда нужно добавить конкретные изменения
- `git add .` - когда все изменения связаны и должны быть в одном коммите

## 5. Что показывает команда `git status` и как меняется ее вывод?

`git status` показывает:
- Измененные, но не добавленные в индекс файлы
- Файлы в индексе, готовые к коммиту
- Неотслеживаемые файлы

**Изменения вывода:**
- После изменений: показывает modified files
- После `git add`: показывает changes to be committed
- После `git commit`: показывает "nothing to commit, working tree clean"

## 6. Для чего нужна команда `git remote add origin URL` и что означает "origin"?

Команда связывает локальный репозиторий с удаленным репозиторием.

**"origin"** - это стандартное имя для удаленного репозитория-источника. Это псевдоним для URL удаленного репозитория.

## 7. Зачем нужен флаг `-u` в команде `git push -u origin main`?

Флаг `-u` (или `--set-upstream`) устанавливает связь между локальной веткой и удаленной веткой. После этого можно использовать просто `git push` без указания репозитория и ветки.

## 8. Что такое ветка по умолчанию и какая у вас создалась?

**Ветка по умолчанию** - это основная ветка проекта, куда по умолчанию попадают изменения.

У меня создалась ветка **main**. Это важно потому, что:
- Это основная ветка разработки
- На нее обычно настроены процессы CI/CD
- Она считается стабильной версией проекта

## 9. Как просмотреть историю коммитов и какую информацию вы видите?

**Команда:** `git log`

**Информация:**
- Хеш коммита (уникальный идентификатор)
- Автор коммита
- Дата и время коммита
- Сообщение коммита

## 10. Что произойдет, если сделать изменения в файле, но не выполнить `git add` перед `git commit`?

Изменения не попадут в коммит. Git создаст коммит только из тех файлов, которые были добавлены в индекс с помощью `git add`. Непроиндексированные изменения останутся в рабочей директории.

## Семантическая верстка

### 1. Чем семантическая верстка отличается от несемантической и почему она важна?

**Семантическая верстка** использует HTML-теги, которые несут смысловое значение (`<header>`, `<article>`, `<nav>` и т.д.).

**Несемантическая верстка** использует в основном `<div>` и `<span>` без конкретного смысла.

**Важность семантической верстки:**
- Улучшает доступность для скринридеров
- Повышает SEO-оптимизацию
- Упрощает чтение и поддержку кода
- Обеспечает лучшую структуру документа

### 2. В каких случаях следует использовать `<article>` вместо `<section>`? Приведите примеры.

**`<article>`** используется для независимого, самодостаточного контента, который можно распространять отдельно:
- Пост в блоге
- Новостная статья
- Комментарий пользователя
- Карточка товара

**`<section>`** используется для группировки тематического контента внутри документа:
- Глава книги
- Введение на странице
- Группа новостей

### 3. Для чего предназначены теги `<header>` и `<footer>` и могут ли они использоваться несколько раз?

**`<header>`** - для вводного контента или навигации в пределах своего родительского элемента.

**`<footer>`** - для заключительного контента (авторство, ссылки, копирайт).

**Да, они могут использоваться несколько раз** - для всего документа и для каждой секции/статьи.

### 4. Какой тег следует использовать для основного контента страницы и почему не стоит использовать `<div>`?

Для основного контента используется **`<main>`**.

**Почему не `<div>`:**
- `<main>` дает четкое указание на основное содержание
- Улучшает навигацию для скринридеров
- Помогает поисковым системам определить главный контент
- Имеет семантическое значение

### 5. В чем разница между `<nav>` и обычным списком ссылок?

**`<nav>`** - семантический тег для основной навигации по сайту.

**Разница:**
- `<nav>` указывает на важный навигационный блок
- Скринридеры распознают его как навигацию
- Поисковые системы придают больше веса ссылкам в `<nav>`

**Обязательно использовать `<nav>`** для основной навигации (меню сайта, оглавление).

### 6. Для чего используется атрибут datetime в теге `<time>` и почему он важен?

**`datetime`** предоставляет машиночитаемую версию времени, в то время как содержимое тега - человекочитаемую.

**Важность:**
- Позволяет поисковым системам понимать даты
- Упрощает извлечение дат программами
- Обеспечает единый формат для обработки

### 7. Какие преимущества дает использование `<figure>` и `<figcaption>`?

**Преимущества:**
- Семантическая связь между изображением и подписью
- Улучшенная доступность для скринридеров
- Лучшее понимание контекста поисковыми системами
- Возможность группировать несколько медиа-элементов с одной подписью

### 8. Когда следует использовать `<aside>` и какой контент в него помещать?

**`<aside>`** используется для контента, косвенно связанного с основным содержанием:

- Боковые панели
- Блоки с дополнительной информацией
- Рекламные баннеры
- Ссылки на related content
- Цитаты

### 9. Как семантическая разметка влияет на SEO и доступность веб-страницы?

**Для SEO:**
- Поисковые системы лучше понимают структуру контента
- Улучшается релевантность страницы запросам
- Повышается вероятность rich snippets

**Для доступности:**
- Скринридеры могут правильно ориентировать пользователей
- Упрощается навигация по странице
- Улучшается пользовательский опыт для людей с ограниченными возможностями

### 10. Какие теги вы использовали для улучшения структуры документа?

В своей работе я использовал:
- **`<header>`** - для шапки сайта
- **`<nav>`** - для основной навигации
- **`<main>`** - для основного контента
- **`<article>`** - для независимого контента
- **`<section>`** - для тематических групп
- **`<aside>`** - для дополнительной информации
- **`<footer>`** - для подвала
- **`<figure>`** и **`<figcaption>`** - для изображений с подписями

**Польза:**
- Пользователи быстрее находят нужный контент
- Поисковые системы лучше индексируют страницу
- Упрощается поддержка и обновление кода

# Ответы на вопросы для самоконтроля

## 1. Разница между селектором по классу и по ID
- **Специфичность**: ID имеет специфичность (1,0,0), класс - (0,1,0)
- **Повторное использование**: Классы можно применять к множеству элементов, ID должен быть уникальным на странице

## 2. Как работает каскад в CSS
Порядок применения стилей при конфликте:
1. Важность (!important)
2. Специфичность селектора
3. Порядок в коде (последнее правило имеет приоритет)

## 3. Расчет специфичности
- `ul li.important` = (0,1,2)
- `#site-title` = (1,0,0)
- `.content > .featured-post` = (0,2,0)

## 4. Наследуемые и ненаследуемые свойства
**Наследуемые**: `color`, `font-family`, `line-height`
**Ненаследуемые**: `margin`, `padding`, `border`

## 5. Разница между > и пробелом
- `A > B` - выбирает непосредственных потомков
- `A B` - выбирает всех потомков любого уровня вложенности

## 6. Псевдоклассы :hover и :nth-child()
- `:hover` - стили при наведении (кнопки, ссылки)
- `:nth-child()` - выбор элементов по порядку (полосатые таблицы, списки)

## 7. Правило !important
Переопределяет все другие правила. Используется осторожно, так как усложняет поддержку кода.

## 8. Псевдоэлементы ::before и ::after
Добавляют декоративный контент: иконки, цитаты, оформление. Могут содержать текст, изображения, быть использованы для создания фигур.

## 9. Порядок применения при одинаковой специфичности
Браузер применяет последнее объявление в коде.

## 10. Инструменты разработчика
Показывают примененные стили, специфичность, наследование, позволяют экспериментировать с изменениями в реальном времени.

# Ответы на вопросы по Flexbox

## 1. Основное свойство CSS для Flexbox
Основное свойство - `display: flex`. После его применения дочерние элементы становятся flex-элементами и выстраиваются вдоль основной оси.

## 2. Разница между justify-content и align-items
- `justify-content` - выравнивание вдоль основной оси (горизонтально по умолчанию)
- `align-items` - выравнивание вдоль поперечной оси (вертикально по умолчанию)

## 3. Свойство flex-direction
Определяет направление основной оси. Значения:
- `row` (по умолчанию) - слева направо
- `row-reverse` - справа налево  
- `column` - сверху вниз
- `column-reverse` - снизу вверх

## 4. Сокращенная запись flex: 1
Эквивалентно:
- `flex-grow: 1` - элемент может увеличиваться
- `flex-shrink: 1` - элемент может уменьшаться
- `flex-basis: 0` - базовый размер 0

## 5. Свойство flex-wrap
Определяет, могут ли элементы переноситься на новую строку. Используется когда элементы не помещаются в одну строку.

## 6. Разница между align-content и align-items
- `align-items` - выравнивание отдельных элементов в строке
- `align-content` - выравнивание строк в многорядном flex-контейнере

## 7. Свойство order
Определяет порядок отображения элементов. Полезно для изменения порядка без изменения HTML.

## 8. Осевые линии в Flexbox
- Основная ось (main axis) - определяется flex-direction
- Поперечная ось (cross axis) - перпендикулярна основной

## 9. Адаптивность без медиа-запросов
Использование `flex-wrap: wrap` и `flex-basis` с min/max-width:
```css
.container {
    display: flex;
    flex-wrap: wrap;
}
.item {
    flex: 1 1 200px;
    min-width: 200px;
}

1. **Простота и удобство** - один свойство вместо множества margin-правил
